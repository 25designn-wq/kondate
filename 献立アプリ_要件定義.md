# 献立アプリ 要件定義

## コンセプト

夫婦2人で使うAI献立提案アプリ。
AIが7日分の夕飯を提案し、2台のスマホで確認・決定できる。
使うほど精度が上がる学習データベースが核心。

---

## 開発環境

- HTML + CSS + JS（ビルドツールなし、ライブラリはCDN）
- ローカル確認：`python3 -m http.server 8765`
- Firebase Firestore（リアルタイム同期）
- AI：Gemini 2.0 Flash（無料枠）
- デプロイ：GitHub Pages（Public リポジトリ）

---

## ユーザー構成

- 夫・妻の2人
- 子ども：なぎ（野菜が苦手）

## 画面一覧と遷移

### 画面一覧

| # | 画面名 | 概要 |
|---|--------|------|
| 1 | ホーム | 今週の状況表示・各アクションへの入口 |
| 2 | ユーザー選択 | 夫 / 妻 どちらとして使うか（初回 + 切替可） |
| 3 | 初期設定 | 個人別プロフィール設定 |
| 4 | なぎ設定 | 共通：食べられないもの |
| 5 | 週次アンケート | 今週の気分 + 7日分のモード設定 |
| 6 | 発表・スワイプ画面 | ドラムロール → カード発表 → スワイプ採用/却下 |
| 7 | 今週の献立 | 確定した7日分の一覧・詳細・変更 |
| 8 | 設定 | 初期設定・なぎ設定の編集 |

### 遷移フロー

```
起動
 └─ 初回のみ ──→ ユーザー選択 → 初期設定 → なぎ設定
                                                  ↓
                                               ホーム
                                                  │
                              ┌───────────────────┤
                              ↓                   ↓
                         週次アンケート入力    今週の献立を見る
                         （各自・非同期）
                              ↓
                    2人入力済みで「会議を始める」が解放
                              ↓
                    ドラムロール → カード発表 → スワイプ × 7日
                              ↓
                         献立確定 🎉 → 今週の献立へ
```

### ホームの状態管理

| 状態 | 表示 |
|------|------|
| 未入力 | 「アンケートを入力する」 |
| 自分だけ入力済み | 「パートナーの入力待ち…」 |
| 2人とも入力済み | 「会議を始める」（解放） |
| 献立確定済み | 「今週の献立を見る」 |

## 機能詳細

### 初期設定（個人別：夫 / 妻 それぞれ）

- 嫌いな食材・アレルギー
- 好きなジャンル（和食・洋食・中華・エスニックなど）
- 食事の目標（なし / ダイエット / たくさん食べたい など）
- 利用可能な調理器具（オーブン・圧力鍋など）

### なぎ設定（共通）

- 食べられないもの・嫌いなもの

### 週次アンケート（個人別・非同期入力）

- 今週の気分（フリーテキスト、一言）
- 食べたいもの（任意）
- 避けたいもの（任意）
- 7日分のモード設定（提案必要 / 外食でいい / 出来合いでいい / スキップ）

2人分の入力が揃ったらAIが7日分を一発提案。

### 献立カードのスワイプ操作

| 操作 | 結果 | 演出 |
|------|------|------|
| 右スワイプ | 採用 ✅ | パチパチパチ + 紙吹雪 |
| 左スワイプ | 不採用（理由なし）❌ | ブーブー |
| 下スワイプ | 不採用（理由あり）📋 | 理由ボタン選択 → ブーブー |

**却下理由ボタン（下スワイプ後）**
- 😤 このメニューが嫌い → 二度と提案しない
- 🛒 材料がない → また提案してよい
- 😮‍💨 作るのが面倒 → 頻度を下げる

### 発表演出（会議画面）

- ドラムロール → スポットライト → 各日のカードがドーンと登場（月→日の順）
- 採用確定で紙吹雪
- デザインはクール・シンプル。ギミックだけ派手に

### 子ども対応（なぎ）

- 献立になぎの食べられないものが含まれる場合、カードに「なぎはどうする？」欄を表示
- AIは子供の代替案を自動提案しない。親が自由にメモするだけ

## AIの提案設計

### プロンプトに毎回渡すもの

1. 夫・妻それぞれの初期設定（プロフィール）
2. なぎの食べられないもの
3. 今週のアンケート（2人分）
4. 学習データベース（直近3ヶ月分）
5. 今日の日付（季節・旬の判断に使用）

### 季節対応

- 日付から季節を判断し、旬の食材・温冷の偏りを考慮する
- 旬食材を使う場合は献立カードに短いコメントを追加
  - 例：「🍂 旬コメント：9〜10月が脂の乗ったさんまの旬。今が一番おいしい時期です」

### 学習データベースの蓄積（Firestoreに保存）

| データ | 内容 | 使われ方 |
|--------|------|----------|
| 却下ログ | メニュー名・却下理由・日付 | 「嫌い」は二度と出さない。「面倒」は頻度を下げる |
| 採用ログ | 採用メニュー・日付 | 似た系統を定期的に提案する |
| 週次アンケート履歴 | 2人の入力・日付 | 季節・時期ごとの傾向を把握 |
| 自由変更ログ | 提案を差し替えた内容 | 差し替え先がヒントになる |

---

## デザイン方針

- **全体**：クール・シンプル・格好いい
- **ギミック**：ドラムロール、スポットライト、紙吹雪、スワイプ効果音などはあえて派手に
- **フォント・カラー**：モノトーン寄り、アクセントカラー1色程度
- **対象デバイス**：スマートフォン（430×820px 基準）
