# こんだて — 実装メモ

夫婦で夕飯の献立をAIに決めてもらうスマホ向けWebアプリ。
ビルドツールなし、CDN読み込みのみの静的SPAとして実装。

---

## 技術スタック

| 要素 | 選択 | 理由 |
|---|---|---|
| UI | Vanilla JS + カスタム `h()` | ビルド不要、ES modules のみ |
| スタイル | CSS変数 + 手書きCSS | 依存ゼロ、白ベース |
| DB/同期 | Firebase Firestore (Spark無料枠) | 2台スマホのリアルタイム同期 |
| AI | Gemini 2.0 Flash REST API | 無料枠・週1回程度の呼び出し |
| ホスト | GitHub Pages (Public) | 無料・SSL付き |
| フォント | Outfit + Zen Kaku Gothic New (Google Fonts CDN) | |

## Firestore データ構造

合言葉（householdId）をルートキーとして、夫婦のデータを1ツリーにまとめる。

```
households/{合言葉}/
├── meta/
│   ├── profiles   → { husband: {genres,dislikes,goal,tools}, wife: {...} }
│   ├── nagi       → { banned: ['エビ', ...] }
│   └── learning   → { rejected: [...], accepted: [...], changes: [...] }
├── surveys/
│   └── {週ID}     → { husband: {mood,want,avoid,done}, wife: {...} }
└── menus/
    └── {週ID}     → { days: [{weekday,name,reason,seasonal,kidsNote}×7],
                        confirmedBy, confirmedAt }
```

週IDの形式: `2026-W25`（月曜始まりのISO週番号。`state.currentWeekId()` で取得）

### 学習DB（learning）の設計

- `rejected[]`: `{name, reason, rejectedAt}` — reason は `hate` / `stock` / `heavy`
  - `hate` → 二度と提案しないリスト（プロンプトに"絶対出さない"で渡す）
  - `heavy` → 頻度を下げるリスト
  - `stock` → 買い物的な問題（特殊扱いなし）
- `accepted[]`: `{name, acceptedAt}` — 採用された料理（似た系統を時々提案）
- `changes[]`: `{from, to}` — 差し替え履歴（好みのヒント）

プロンプトに渡す件数上限: rejected×60, accepted×40, changes×30（トークン肥大化防止）

## 会議画面（meeting.js）の詳細

アプリの核心。フローは以下の順で進む。

1. **loadData** — Firestore からプロフィール・なぎ・アンケート・学習DBを取得
2. **generateMenu** — Gemini APIを呼び出し（キーなしの場合はデモデータを使用）
3. **showIntro** — 「会議をはじめる」ボタン + ドラムロールアニメーション
4. **startCards** — 7日分のカードをスワイプで採否決定
5. **saveResult** — 確定したメニューをFirestoreへ保存 + 学習DBを更新

### スワイプ操作（Pointer Events API）

| 方向 | 動作 | 効果音 |
|---|---|---|
| 右 (dx > 90px) | 採用 | `clap()` — 3連バースト |
| 左 (dx < -90px) | 却下（理由なし） | `buzz()` — 2×矩形波 |
| 下 (dy > 90px) | 却下 + 理由選択オーバーレイ | `buzz()` |

却下理由: 「嫌い（hate）」「買い物が大変（stock）」「作るのが面倒（heavy）」

### エフェクト

- `confetti(amount)` — `#fx-layer` に `.confetti-piece` divを注入するCSSアニメーション
- `drumrollSound()` — Web Audio APIでノイズ + 最後にキック
- 7日カード全採否完了 → 確定済み献立の全カード表示 + 紙吹雪

## セキュリティ方針

- **Firebase config（apiKey等）** はソースに直書き可（Firestoreのセキュリティルールで保護する前提）
- **Gemini APIキー** はlocalStorageに保存し、ソースには絶対に書かない。設定画面から各端末で入力。
- **Firestoreルール**: `allow read, write: if true` — 夫婦2人での小規模利用のため許容
- **GitHubリポジトリ**: Public公開（GitHub Pagesのため）。Geminiキーは入らないので問題なし

## DOM設計

すべての画面は `h()` ヘルパーで組み立てる（ReactのJSXに相当するが、ビルド不要）。

```js
// 使い方
import { h, topbar, chipGroup, tagInput } from '../dom.js';

const el = h('div', { class: 'card', onclick: fn },
  h('p', {}, 'テキスト'),
  child,
);
```

`chipGroup` と `tagInput` は要素に `.getValue()` メソッドが生え、現在値を返す。

## Firebase設計

`firebase.js` の主要API:

| 関数 | 説明 |
|---|---|
| `read(path, fallback)` | getDoc + 8秒タイムアウト。失敗時はfallback返却 |
| `write(path, data)` | setDoc with merge:true |
| `watchDoc(path, cb)` | onSnapshot — unsubscribe関数を返す |
| `getProfiles/setProfile` | プロフィール読み書き |
| `getLearning/setLearning` | 学習DB読み書き |
| `getSurvey/setSurvey` | 週次アンケート読み書き |
| `getMenu/setMenu` | 確定献立読み書き |

ホーム画面は `watchDoc` でリアルタイム更新し、`MutationObserver` で画面破棄時に購読解除（メモリリーク防止）。

## Gemini APIキー設定手順

1. [Google AI Studio](https://aistudio.google.com/) でAPIキーを作成
2. アプリを開く → 右上 ⚙️ → 「Gemini API キー」欄に貼り付けて保存
3. 会議画面で「会議をはじめる」を押すとAI生成が走る

キーは端末のlocalStorageにのみ保存。もう一方のスマホにも同じ手順で設定が必要。
